# Internationalization (i18n) — Bahasa Indonesia / English

Bilingual UI. Default locale is **Indonesian (`id`)**; **English (`en`)** is the alternate. Users switch with the **ID / EN** toggle in the sidebar footer and top header (and on the login/guest pages).

## How it works
- **Choice is session-stored.** `POST /locale` (`locale.update`, `LocaleController@update`) validates `locale ∈ {id, en}` and puts it in the session. Works for guests and authenticated users.
- **Backend applies it per request.** `App\Http\Middleware\SetLocale` (registered in `bootstrap/app.php`, *before* `HandleInertiaRequests`) reads `session('locale')` (default `id`) and calls `app()->setLocale()`, so Laravel validation/lang strings match too.
- **Frontend reads it as a shared prop.** `HandleInertiaRequests::share()` exposes `locale => app()->getLocale()` to every page.

## Frontend usage
- Dictionaries: `resources/js/i18n/id.js` and `en.js` (nested objects, **keep keys in sync**). `index.js` exports `translations`, `SUPPORTED_LOCALES`, `DEFAULT_LOCALE`, and the `useTranslation()` hook.
- In a component:
  ```jsx
  import { useTranslation } from '@/i18n';
  const { t, locale } = useTranslation();
  <span>{t('nav.dashboard')}</span>   // dot-path key; falls back to id, then the key
  ```
- Switcher: `resources/js/Components/LanguageSwitcher.jsx` (`variant="dark"` for the dark sidebar, `"light"` elsewhere). POSTs to `locale.update` with `preserveState: false` so the reload re-shares the new `locale`.

## Scope so far
- Fully translated:
  - **App shell** — `Layouts/AuthenticatedLayout.jsx` (sidebar nav for every role, mobile bottom nav, header, footer, role labels, logout modal, tooltips) and `Layouts/GuestLayout.jsx`.
  - **`Pages/Auth/Login.jsx`** (namespace `auth`) — standalone page, has its own `LanguageSwitcher` top-left.
  - **`Pages/Dashboard.jsx`** (namespace `dash`) — admin/cashier/penitip views, charts legends/tooltips, tables; date label follows `locale`. Note: uses `const { t: tr }` because the recent-transactions `.map((t) => …)` already binds `t`.
- **Not yet translated:** the remaining page bodies (`Pages/**`) still contain hardcoded Indonesian. Migrate them incrementally: `const { t } = useTranslation()`, add keys under a page namespace in both dictionaries, replace the literals with `t(...)`.

## Gotchas
- Add every new key to **both** `id.js` and `en.js`. Missing keys fall back to Indonesian, then to the raw key string.
- `SetLocale` must stay ordered before `HandleInertiaRequests` in `bootstrap/app.php`, or the shared `locale` prop lags one request behind.
- `SUPPORTED` locales are declared in `SetLocale::SUPPORTED` (PHP) and `SUPPORTED_LOCALES` (JS) — update both when adding a language.
